# Contributing to Impactly

We welcome contributions! To ensure a smooth collaboration, please follow our guidelines.

## Branching Strategy
- **main**: The stable production-ready branch.
- **develop**: Integration branch for features.
- **feature/[feature-name]**: Individual feature branches.

## Commit Message Convention
We follow a simplified imperative style for commit messages:
- Use the present tense ("Add feature" not "Added feature")
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
- Be descriptive but concise.

## Pull Request Process
1. Fork the repository.
2. Create your feature branch from `develop`.
3. Ensure your code follows the project's style guidelines.
4. Submit a Pull Request to the `develop` branch.
5. At least one team member must review and approve the PR before merging.
